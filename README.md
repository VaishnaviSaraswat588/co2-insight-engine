# 🌍 CO2 Emissions Analysis — Data Cleaning, EDA & Predictive Modeling

An end-to-end data analyst / data science portfolio project built on a **messy, real-world-style CO2 emissions dataset**. This project walks through every stage of the analytics lifecycle: exploration, cleaning, preprocessing, EDA, visualization, machine learning, and physics-inspired forecasting — with an executive summary at the end.

## 🗂️ Dataset Description

The raw dataset contains CO2 emission records across **5 countries** (China, Germany, India, UK, USA), multiple **sectors** (Power, Industry, Transport, Residential), and **fuel types** (Coal, Oil, Natural Gas, Renewables) from 1990–2023.

| Column | Description |
|---|---|
| `Emission_ID` | Unique record identifier |
| `Country` | Country name |
| `Sector` | Emitting sector (Power, Industry, Transport, Residential) |
| `Fuel_Type` | Fuel source (Coal, Oil, Natural Gas, Renewables) |
| `Year` | Year of record (1990–2023) |
| `CO2_Emissions_MMt` | CO2 emissions in million metric tons |
| `Population_Million` | Country population (millions) |
| `GDP_Billion_USD` | GDP in billion USD |

### ⚠️ Data Quality Issues (by design)

This dataset was deliberately made "dirty" to simulate real-world data problems:

- Inconsistent text casing — `usa`, `USA`, `china ` (with trailing whitespace)
- A disguised missing value — the literal string `"unknown"` inside a numeric column
- **15 completely blank rows**
- **20 exact duplicate rows**
- **GDP outliers** — data-entry-style errors (values >$2000B against a normal range of ~$130–700B)
- Missing values scattered across `Sector`, `Fuel_Type`, `Population_Million`, and `GDP_Billion_USD`

Handling these issues is the core of the cleaning stage — see [`co2_emissions_analysis.py`](./co2_emissions_analysis.py) for the full logic.

## 🔬 Methodology

### 1. Data Exploration
Shape, dtypes, missing-value audit, duplicate detection, and unique-value inspection to identify every quality issue before touching the data.

### 2. Data Cleaning
- Standardized text casing and whitespace across `Country`, `Sector`, `Fuel_Type`
- Converted the disguised `"unknown"` string to a proper `NaN`, then to numeric
- Removed fully blank and duplicate rows
- Flagged and nulled GDP outliers using the **IQR rule**
- Imputed missing numeric values with **group-wise medians** (by country), and categorical gaps with an explicit `"Unknown"` label

### 3. Data Preprocessing
- Feature engineering: `Emissions_per_Capita`, `Carbon_Intensity` (CO2 per unit GDP), `Decade`
- `ColumnTransformer` pipeline: `StandardScaler` for numeric features, `OneHotEncoder` for categorical features
- Train/test split (80/20)

### 4. Exploratory Data Analysis (EDA)
Group-by summaries across country, sector, fuel type, and decade; correlation analysis between emissions, population, GDP, and derived metrics.

### 5. Data Visualization
9 charts covering trends over time, sector/fuel breakdowns, correlation heatmap, model comparison, feature importance, and PCA projection.

### 6. Machine Learning
Five regression models trained and compared to predict `CO2_Emissions_MMt`:
- Linear Regression
- Ridge Regression
- Random Forest Regressor
- Gradient Boosting Regressor
- Support Vector Regression (RBF kernel)

Evaluated on **MAE**, **RMSE**, and **R²**, with Random Forest feature importances to interpret key drivers.

### 7. Physics-Inspired Modeling & Forecasting
Two growth-curve models — borrowed directly from physics — fitted to the emissions trend using `scipy.curve_fit`:
- **Exponential growth model** (`dy/dt = ky`) — the same law that governs radioactive decay and unconstrained population growth
- **Logistic growth model** (`dy/dt = ky(1 - y/L)`) — an "S-curve" that saturates at a ceiling, used to model resource-limited growth

Also includes a **PCA projection**, framed as the data-science analogue of finding principal axes of inertia in mechanics.

### 8. Executive Summary
Auto-generated, plain-English key findings printed at the end of the script — ready to present to a non-technical stakeholder.

## 📊 Sample Insight

> ⚠️ **Honest finding, not hidden:** all five regression models returned **negative R² scores** on this dataset — meaning none of them beat simply predicting the average. With only ~1,000 rows and effectively noise-driven emissions values, this is a legitimate and useful conclusion: **the target variable shows no strong learnable relationship with the available features**, which shifts the story toward descriptive/trend analysis rather than predictive modeling. Naming this limitation explicitly is itself good analyst practice.

## ▶️ How to Run

1. Clone this repository and install dependencies:
   ```bash
   pip install pandas numpy matplotlib seaborn scikit-learn scipy
   ```
2. Place `co2_emissions_dirty_dataset.csv` in the same directory as the script.
3. Run the pipeline:
   ```bash
   python co2_emissions_analysis.py
   ```
4. Check the `outputs/` folder for all generated charts and the cleaned CSV.

> 💡 Prefer an interactive walkthrough? Copy each numbered section of the script into its own Jupyter Notebook cell to see tables and charts render as you go.
